1153 impurity criterion.

1157

1159 ---------- 

1160 tree 

1161 

1162 Returns 

1163 ------- 

1164 

1165 """

1353def compute_adjusted_error(err, n_samples, n_parameters, multiplier=PRUNING_MULTIPLIER): 

1354 """ 

1355 Return the penalized error obtained from `err`, the prediction error at a 

1356 given tree node, by multiplying it by ``(n+p)/(n-p)``. `n` is the number of 

1357 samples at this node and `p` is the number of parameters of the model. 

1358 

1359 According to the original M5 paper, this is to penalize complex models 

1360 used for few samples. 

1361 

1362 Note that if ``n_samples <= n_parameters`` the denominator is zero or negative. 

1363 In this case an arbitrary high penalization factor is used: ``10 * err``. 

1364 

1365 Parameters 

1366 ---------- 

1367 err : float 

1368 The model error for a given tree node. Typically the MAE. 

1369 

1370 n_samples : int 

1371 The number of samples at this node. 

1372 

1373 n_parameters : int 

1374 The number of parameters of the model at this node. 

1375 

1376 Returns 

1377 ------- 

1378 penalized_mae : float 

1379 The penalized error ``err * (n+p)/(n-p)``, or ``10 * err`` if ``n <= p``. 

1380 """ 

1381 # 

1382 if n_samples <= n_parameters: 1382 ↛ 1385line 1382 didn't jump to line 1385, because the condition on line 1382 was never true

1383 # denominator is zero or negative: use a large factor so as to penalize 

1384 # a lot this overly complex model. 

1385 factor = 10.0 # Caution says Yong in his code 

1386 else: 

1387 factor = (n_samples + multiplier * n_parameters) / (n_samples - n_parameters) 

1388 

1389 return err * factor
